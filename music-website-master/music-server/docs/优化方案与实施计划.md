## 三、LangChain4j 音乐 Agent 方案

### 3.1 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| **LLM** | DeepSeek (`deepseek-chat`) | 已配置 API Key, 中文效果好 |
| **嵌入模型** | BGE-small-zh | 本地运行, 中文优化, LangChain4j 原生支持 |

```python
# 核心逻辑（伪代码）
# 1. 从 MySQL 取歌曲列表（前 500 首热门歌即可）
songs = "SELECT s.id, s.name, si.name as singer 
         FROM song s JOIN singer si ON s.singer_id = si.id 
         ORDER BY s.play_count DESC LIMIT 500"

# 2. 对每首歌搜索
for song in songs:
    query = f"{song['singer']} {song['name']} 歌曲 创作背景 歌词含义"
    # 方式A: requests + BeautifulSoup 抓百度百科
    # 方式B: 第三方搜索 API
    content = search_and_extract(query)
    
    # 3. 清洗文本（去HTML标签、广告、短行）
    content = clean(content)
    
    # 4. 保存
    with open(f"rag-docs/{song['id']}.txt", "w", encoding="utf-8") as f:
        f.write(f"{song['name']}|{song['singer']}\n")  # 第一行为元数据
        f.write(content)
```

### Step 5: `DocumentIngestionService.java`

```java
@Service
@Slf4j
public class DocumentIngestionService implements ApplicationRunner {

    @Autowired private EmbeddingModel embeddingModel;
    @Autowired private EmbeddingStore<TextSegment> embeddingStore;

    @Override
    public void run(ApplicationArguments args) {
        // 1. 加载 rag-docs/ 下所有 .txt 文件
        List<Document> docs = FileSystemDocumentLoader
            .loadDocuments("rag-docs/", new TextDocumentParser());

        // 2. 切割文档: 每 500 字一块，重叠 50 字保证语义连贯
        DocumentSplitter splitter = DocumentSplitters.recursive(500, 50);
        List<TextSegment> segments = splitter.splitAll(docs);

        // 3. 批量向量化 + 存储
        List<Embedding> embeddings = embeddingModel.embedAll(
            segments.stream().map(TextSegment::text).collect(Collectors.toList())
        ).content();
        
        for (int i = 0; i < segments.size(); i++) {
            embeddingStore.add(embeddings.get(i), segments.get(i));
        }

        log.info("RAG 知识库构建完成，共 {} 个文档片段", segments.size());
    }
}
```

### Step 8: `AiServiceImpl.java`（重写）

```java
@Service
public class AiServiceImpl implements AiService {

    @Autowired private ChatLanguageModel chatModel;
    @Autowired private MusicTools musicTools;
    @Autowired private SongContentRetriever retriever;

    private final Map<String, ChatMemory> memories = new ConcurrentHashMap<>();

    @Override
    public R chat(ChatRequest request) {
        // 提取用户消息
        String userId = extractUserId(request);
        String userMsg = extractUserMessage(request);

        // 组装 Agent = LLM + Tool + RAG + Memory
        MusicAssistant assistant = AiServices.builder(MusicAssistant.class)
            .chatLanguageModel(chatModel)
            .tools(musicTools)
            .contentRetriever(retriever.build())
            .chatMemoryProvider(id -> memories.computeIfAbsent(id,
                k -> MessageWindowChatMemory.withMaxMessages(20)))
            .build();

        String reply = assistant.chat(userId, userMsg);
        return R.success("成功", Map.of("reply", reply));
    }
}

// Agent 接口定义
interface MusicAssistant {
    @SystemMessage("""
        你是一个专业的音乐助手，名字叫"小音"。你可以：
        1. 使用 searchSongs 搜索歌曲
        2. 使用 searchSongsBySinger 搜索歌手的歌曲
        3. 使用 getRankList 查看排行榜
        4. 使用 getSongLists 浏览歌单
        5. 使用 searchSongListByStyle 按风格搜索歌单
        6. 使用 searchSingers 搜索歌手信息
        7. 使用 recommendSongs 推荐歌曲
        
        关于播放控制：
        - 推荐歌曲时，在回复末尾加上 [PLAY:歌名-歌手名] 标记
        - 用户要求暂停/继续/下一首时，回复 [PAUSE]/[RESUME]/[NEXT]
        
        重要规则：
        - 只回答和音乐相关的问题
        - 使用中文，保持温暖友好的语气
        - 当数据库搜索不到结果时，诚实告知用户
        """)
    String chat(@MemoryId String userId, @UserMessage String message);
}
```
